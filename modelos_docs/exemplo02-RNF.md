# Exemplo de Requisitos Não Funcionais 03

## Sistema: Acadêmico

### RNF01. Segurança
O sistema deve permitir acesso às informações acadêmicas somente a usuários autenticados e autorizados.

### RNF02. Manutenibilidade
O código do sistema deve seguir padrão de organização que facilite correções e evolução futura.

### RNF03. Desempenho
O sistema deve exibir o boletim do aluno em até 2 segundos após a solicitação.

### RNF04. Confiabilidade
O sistema deve garantir consistência dos dados de matrícula, notas e frequência em todas as operações.

### RNF05. Portabilidade
O sistema deve ser acessível em computadores, tablets e celulares por meio do navegador.
