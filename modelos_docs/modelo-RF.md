# Modelo de Requisitos Funcionais

## Instruções

Requisitos funcionais descrevem **o que o sistema deve fazer**.  
Eles representam funcionalidades, serviços e comportamentos esperados pelos usuários.

Escreva os requisitos de forma clara, direta e objetiva.

---

## Exemplo de escrita

- O sistema deve permitir que o aluno realize matrícula em disciplinas.
- O sistema deve permitir que o administrador cadastre novos produtos.
- O sistema deve gerar relatórios de vendas por período.

## Estrutura recomendada

Uma boa forma de escrever é:

**O sistema deve + ação + complemento**

**Exemplos:**

- O sistema deve permitir que o cliente acompanhe o status do pedido.
- O sistema deve registrar a devolução de livros emprestados.
- O sistema deve emitir o histórico escolar do aluno.

---

## Tipos comuns de requisitos funcionais

### Cadastro
Funções de inclusão de dados no sistema.

**Exemplos:**
- cadastrar usuário
- cadastrar produto
- cadastrar disciplina

### Consulta
Funções de visualização ou busca de informações.

**Exemplos:**
- consultar boletim
- consultar estoque
- consultar pedidos

### Atualização
Funções de alteração de dados já existentes.

**Exemplos:**
- atualizar cadastro
- alterar preço de produto
- editar informações de uma turma

### Remoção
Funções de exclusão ou cancelamento.

**Exemplos:**
- excluir usuário
- cancelar pedido
- remover produto do carrinho

### Processamento
Funções em que o sistema executa alguma regra ou operação.

**Exemplos:**
- calcular média final
- gerar relatório
- validar matrícula
- processar pagamento

---

## Orientações

- escreva requisitos que representem funcionalidades reais do sistema
- use verbos de ação, como cadastrar, consultar, atualizar, excluir, emitir, registrar, validar
- evite frases vagas, como “o sistema deve ser eficiente”
- cada requisito funcional deve descrever uma ação ou serviço do sistema
- pense no que o usuário espera fazer ou obter ao usar o sistema
